nova_utils.scripts.predict module

Standalone script for data predictions

Author:

Dominik Schiller <dominik.schiller@uni-a.de>

Date:

06.09.2023

This script performs predictions for annotations in the NOVA database using a provided nova-server module for inference and saves the results to the NOVA-DB.

param –db_host:

The IP address of the NOVA-DB server.

type –db_host:

str, required

param –db_port:

The port number of the NOVA-DB server.

type –db_port:

int, required

param –db_user:

The user to authenticate at the NOVA-DB server.

type –db_user:

str, required

param –db_password:

The password for the NOVA-DB server user.

type –db_password:

str, required

param –dataset:

Name of the dataset. Must match entries in NOVA-DB.

type –dataset:

str, required

param –data_dir:

Path to the NOVA data directory using Windows UNC-Style.

type –data_dir:

str, required

param –sessions:

JSON formatted list of sessions to apply the iterator to.

type –sessions:

json.loads, required

param –data:

JSON formatted string containing dictionaries that describe the data to load.

type –data:

json.loads, required

param –frame_size:

Size of the data frame measured in time. Defaults to None.

type –frame_size:

str, optional

param –start:

Start time for processing measured in time. Defaults to None.

type –start:

str, optional

param –end:

End time for processing measured in time. Defaults to None.

type –end:

str, optional

param –left_context:

Left context duration measured in time. Defaults to None.

type –left_context:

str, optional

param –right_context:

Stride for iterating over data measured in time. If stride is not set explicitly, it will be set to frame_size. Defaults to None.

type –right_context:

str, optional

param –stride:

Stride for iterating over data measured in time. If stride is not set explicitly, it will be set to frame_size. Defaults to None.

type –stride:

str, optional

param –add_rest_class:

Whether to add a rest class for discrete annotations. Defaults to True.

type –add_rest_class:

str, optional

param –fill_missing_data:

Whether to fill missing data. Defaults to True.

type –fill_missing_data:

str, optional

param –cml_dir:

CML-Base directory for the NOVA-Server module.

type –cml_dir:

str, optional

param –opt_str:

JSON formatted string containing dictionaries with key-value pairs, setting the options for a NOVA-Server module.

type –opt_str:

str, optional

param –trainer_file_path:

Path to the trainer file using Windows UNC-Style. (required)

type –trainer_file_path:

str

returns:

None

Example

>>> python predict_annotations.py --dataset "test" --db_host "127.0.0.1" --db_port "37317" --db_user "my_user" --db_password "my_password" --trainer_file_path "test\test_predict.trainer" --sessions "["test_session_1", "test_session_2"]" --data "[{"src": "db:anno", "scheme": "transcript", "annotator": "test", "role": "testrole"}]" --frame_size "0" --left_context "0" --right_context "0" --job_i_d "test_job" --opt_str "num_speakers=2;speaker_ids=testrole,testrole2" --cml_dir "./cml" --data_dir "./data" --log_dir "./log" --cache_dir "./cache" --tmp_dir "./tmp"
